\section{Cuestionario Previo}

\subsection{Indique la función del protocolo IP}

El protocolo IP, Internet Protocol, cumple la función de direccionamiento lógico y entrega de paquetes entre equipos de una misma red o de redes diferentes. Cada host se identifica mediante una dirección IP y los paquetes se envían desde una dirección origen hacia una dirección destino. IP permite que los routers decidan por dónde reenviar los paquetes, aunque no garantiza por sí mismo entrega, orden ni corrección de errores; esas funciones pueden ser asumidas por protocolos superiores como TCP.

\subsection{¿Cuál es la función de una calculadora IP?}

Una calculadora IP permite verificar los datos principales de una red o subred: máscara decimal, prefijo CIDR, dirección de red, dirección de broadcast, rango de hosts válidos, número de hosts disponibles y cantidad de subredes generadas. Es una herramienta útil para comprobar los cálculos manuales de subneteo y evitar asignar direcciones de red, broadcast o direcciones fuera del rango correspondiente.

\subsection{¿Cuál es la función de la máscara en IPv4?}

La máscara en IPv4 separa la porción de red y la porción de host de una dirección IP. Los bits en 1 identifican la red y los bits en 0 identifican al host. Mediante una operación AND entre la dirección IP y la máscara se obtiene la dirección de red; con ese resultado un host determina si el destino está en su misma subred o si necesita un router para comunicarse.

\subsubsection{Operación AND entre 192.168.150.10 y 192.168.150.12}

Con máscara \ip{/25}, ambas direcciones pertenecen a \ip{192.168.150.0/25}:

\[
\ip{192.168.150.10} \; AND \; \ip{255.255.255.128} = \ip{192.168.150.0}
\]
\[
\ip{192.168.150.12} \; AND \; \ip{255.255.255.128} = \ip{192.168.150.0}
\]

Con máscara \ip{/26}, ambas direcciones también pertenecen a \ip{192.168.150.0/26}:

\[
\ip{192.168.150.10} \; AND \; \ip{255.255.255.192} = \ip{192.168.150.0}
\]
\[
\ip{192.168.150.12} \; AND \; \ip{255.255.255.192} = \ip{192.168.150.0}
\]

Por lo tanto, pueden conectarse con máscara de 25 o 26 bits, siempre que estén en el mismo medio físico y no exista una restricción adicional.

\subsubsection{Operación AND entre 192.168.150.10 y 192.168.150.200}

Con máscara \ip{/25}, los resultados son diferentes:

\[
\ip{192.168.150.10} \; AND \; \ip{255.255.255.128} = \ip{192.168.150.0}
\]
\[
\ip{192.168.150.200} \; AND \; \ip{255.255.255.128} = \ip{192.168.150.128}
\]

No pueden conectarse directamente porque pertenecen a subredes distintas. En cambio, con máscara \ip{/24}, ambas direcciones pertenecen a la red \ip{192.168.150.0/24}:

\[
\ip{192.168.150.10} \; AND \; \ip{255.255.255.0} = \ip{192.168.150.0}
\]
\[
\ip{192.168.150.200} \; AND \; \ip{255.255.255.0} = \ip{192.168.150.0}
\]

Por eso, con \ip{/24} sí pueden comunicarse directamente.

\section{Cuestionario Final}

\subsection{¿A qué se denomina dominio de colisión y dominio de broadcast y cómo se soluciona?}

Un dominio de colisión es el conjunto de dispositivos que comparten un mismo medio físico donde dos transmisiones simultáneas pueden interferir. Se reduce usando switches, porque cada puerto del switch crea un dominio de colisión separado. Un dominio de broadcast es el conjunto de equipos que reciben tramas broadcast de capa 2. Se reduce mediante subredes, VLANs y routers o equipos de capa 3.

\subsection{¿Por qué las subredes 1 y 4 de la Tabla 1 están consignadas como no válidas?}

Según el enfoque tradicional de subneteo classful, no se usa la primera subred porque puede confundirse con la red base, ni la última porque puede confundirse con el broadcast del bloque principal. Por eso, al tomar 2 bits se generan 4 subredes, pero se consideran válidas solamente $2^2 - 2 = 2$: la subred 2 y la subred 3. En redes modernas con CIDR muchos equipos permiten usar también la primera y la última, pero la práctica utiliza el criterio clásico.

\subsection{¿Cómo se implementan subredes utilizando direcciones IP de clase A y B? Muestre un ejemplo}

En clase A la máscara por defecto es \ip{/8}; se toman bits de la porción de host para formar subredes. Por ejemplo, la red privada \ip{10.0.0.0/8} puede dividirse en subredes \ip{/16}: \ip{10.1.0.0/16}, \ip{10.2.0.0/16} y \ip{10.3.0.0/16}.

En clase B la máscara por defecto es \ip{/16}. Por ejemplo, \ip{172.16.0.0/16} puede dividirse en subredes \ip{/24}: \ip{172.16.1.0/24}, \ip{172.16.2.0/24} y \ip{172.16.3.0/24}. El procedimiento es el mismo: aumentar el prefijo para crear más subredes y reducir el número de hosts por subred.

\subsection{Análisis para subredes con IPs de clase C robando 4 y 5 bits}

Una red clase C tiene máscara por defecto \ip{/24}. Si se toman 4 bits, el prefijo resultante es \ip{/28}, con máscara \ip{255.255.255.240}. Se generan $2^4 = 16$ subredes, tradicionalmente 14 válidas, y cada una tiene $2^4 - 2 = 14$ hosts útiles.

Si se toman 5 bits, el prefijo resultante es \ip{/29}, con máscara \ip{255.255.255.248}. Se generan $2^5 = 32$ subredes, tradicionalmente 30 válidas, y cada una tiene $2^3 - 2 = 6$ hosts útiles.

\subsection{¿Por qué no se pueden crear subredes con IPs de clase C robando 1 bit o 7 bits?}

Si se toma 1 bit, se generan $2^1 = 2$ subredes, pero con la regla tradicional de subredes válidas se obtiene $2^1 - 2 = 0$ subredes utilizables. Si se toman 7 bits, queda solo 1 bit para hosts; por lo tanto, cada subred tendría $2^1 - 2 = 0$ hosts válidos. En ambos casos, el resultado no es útil para crear subredes funcionales bajo el criterio clásico.

\subsection{¿Por qué se dice que IPv4 es fullclass?}

El término más usado es direccionamiento \textit{classful}. IPv4 originalmente se organizaba en clases A, B y C, donde la máscara por defecto se deducía por el primer octeto de la dirección: clase A \ip{/8}, clase B \ip{/16} y clase C \ip{/24}. Posteriormente se adoptó CIDR, que permite usar prefijos variables como \ip{/27}, \ip{/29} o \ip{/23}, sin depender estrictamente de la clase original.

\subsection{Describa el concepto de superneteo, dé un ejemplo}

El superneteo consiste en agrupar varias redes contiguas en una red mayor usando un prefijo más corto. Se emplea para resumir rutas y reducir el tamaño de las tablas de enrutamiento. Por ejemplo, las redes \ip{192.168.0.0/24}, \ip{192.168.1.0/24}, \ip{192.168.2.0/24} y \ip{192.168.3.0/24} pueden resumirse como \ip{192.168.0.0/22}. Esta superred cubre desde \ip{192.168.0.0} hasta \ip{192.168.3.255}.

\section{Conclusiones}

\begin{enumerate}
    \item El subneteo permite dividir una red grande en redes más pequeñas y administrables.
    \item La máscara de subred define qué bits pertenecen a la red y qué bits pertenecen al host.
    \item La operación AND entre IP y máscara permite determinar si dos hosts pertenecen a la misma subred.
    \item Los switches no enrutan entre subredes; para comunicar subredes distintas se requiere un router o un equipo de capa 3.
    \item Al aumentar los bits de subred, aumenta el número de subredes, pero disminuye la cantidad de hosts disponibles por subred.
    \item En el enfoque tradicional classful se evita usar la primera y la última subred, aunque CIDR moderno puede permitirlo.
    \item VLSM aprovecha mejor el espacio IPv4 porque permite crear subredes de distinto tamaño según la necesidad real de hosts.
    \item No se deben asignar a hosts las direcciones de red ni de broadcast.
    \item Los dominios de broadcast se controlan mediante segmentación lógica, como subredes o VLANs.
    \item El superneteo realiza el proceso inverso al subneteo: agrupa redes contiguas para simplificar el enrutamiento.
\end{enumerate}
